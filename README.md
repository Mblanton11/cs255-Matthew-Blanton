Project Summary

The DriverPass project focused on designing a system for a company that helps students prepare for their driving tests. The client wanted a system that would allow customers to register, schedule lessons, take practice tests, and manage their accounts while giving staff the tools to manage appointments and user information.

What I Did Well

I think I did a good job identifying the client's requirements and turning them into clear UML diagrams and system documentation. I also made sure the design addressed the needs of both customers and employees.

What I Would Revise

If I could revise one part of the project, I would improve my UML diagrams by adding more detail and making some of the relationships between objects clearer. This would make the overall design easier to understand.

Understanding User Needs

I focused on the client's requirements and considered how each type of user would interact with the system. Designing with the user in mind is important because it helps create a system that is easy to use, reliable, and meets the needs of everyone involved.

My Design Approach

When designing software, I like to start by understanding the requirements before creating diagrams and planning the system. In the future, I will continue using UML diagrams, gathering user feedback, and reviewing requirements throughout the design process to build more effective systems.
